[ How it works ]

Three commands to parallel agents

No configuration files. No YAML. Just git, tmux, and a single binary.

1

Initialize in your repo

Run pit init to create a .pit/ directory with a local SQLite database. It auto-adds itself to .gitignore.

2

Create tasks

Run pit new fix-login-bug â€” pit creates a git branch, checks out a worktree, and records the task. Add a prompt, link an issue, or pick an agent.

3

Launch the dashboard

Run pit to open the TUI. Press Enter on any task to drop into its agent session. Press Ctrl-b d to detach and switch to another. All agents keep running.

[ CLI ]

Full command reference

Everything is available from the command line. The TUI is optional.

pit Launch the interactive TUI dashboard
pit init Initialize pit in the current git repository
pit new <name> Create a new task with its own branch and worktree. Flags: -p prompt, -i issue URL, -a agent
pit list List all tasks with their status and branch
pit status Show live task status with automatic reaping of finished agents
pit run <name> Run a task in a detached tmux session (background mode)
pit stop <name> Kill a running task's tmux session
pit diff <name> Show the diff between a task's worktree and the main branch
pit delete <name> Remove a task, its worktree, and its branch
